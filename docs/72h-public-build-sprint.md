# 72-Hour Public Build Sprint Plan

## Goal
Ship visible progress every day for 72 hours, build trust in execution speed, and convert attention into waitlist signups.

## Sprint Window
- Day 1: Foundation + baseline feature shipped
- Day 2: Core UX upgrade + user signal loop
- Day 3: Polish + launch-ready proof + hard CTA push

## Public Proof Rhythm
- 1 morning kickoff post (what we are building today)
- 1 evening proof post (what shipped, with artifact)
- 1 short-form recap clip/screenshot set

## Daily Proof Requirements
Each day must include at least 2 of the following:
1. Commit diff screenshot or PR link
2. Before/after product screenshot or screen recording
3. Metrics snapshot (load time, conversion, usage, bug count)
4. User feedback screenshot (with sensitive details redacted)

## Post Timing (PT)
- Morning kickoff: 9:00 AM
- Evening proof: 6:30 PM
- Waitlist CTA reshare: 8:30 PM

## Deliverables by Day
### Day 1
- Public scope and success criteria
- First shippable improvement in production/staging
- Initial waitlist CTA asset set published

### Day 2
- Usability or feature depth improvement
- Mid-sprint lessons learned post
- Updated CTA variant based on engagement data

### Day 3
- Final sprint demo thread
- Outcome summary with wins/misses/next steps
- Strong waitlist close with urgency messaging

## Success Metrics
- 3 consecutive daily proof posts completed
- Minimum 1 CTA post/day
- Waitlist conversion uplift vs baseline (track with UTM links)
- 3+ direct user replies/DMs indicating interest or feedback

## Execution Checklist
- [ ] Draft all posts before day starts
- [ ] Capture artifacts while building (not after)
- [ ] Log links + metrics in sprint tracker
- [ ] Reply to comments within 2 hours
- [ ] End of day: publish proof + next day teaser
